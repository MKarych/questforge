# План: Каталог игр с вкладками + dropdown в Header

## 1. Изменения в Header (`apps/web/src/components/ui/Header.tsx`)

### 1.1. Убрать "Архив игр" из `mainNavItems`
- Удалить строку `{ label: 'Архив игр', href: '/games/archive', roles: ['PLAYER', 'ORGANIZER', 'ADMIN'] }`

### 1.2. Добавить dropdown "Каталог игр" для авторизованных пользователей
- Для **GUEST** (неавторизованных) — оставить простую ссылку `/games` как сейчас
- Для **авторизованных** (PLAYER, ORGANIZER, ADMIN) — заменить ссылку на `<DropdownNav>` с пунктами:
  - `Актуальные` → `/games`
  - `Архив игр` → `/games/archive`

### 1.3. Обновить мобильное меню
- Убрать "Архив игр" из `visibleMainNav`
- В секции "Основное" для авторизованных добавить подпункты "Актуальные" и "Архив игр" под "Каталог игр" (или просто оставить как отдельные ссылки)

## 2. Изменения на странице `/games` (`apps/web/src/app/games/page.tsx`)

### 2.1. Добавить вкладки (tabs)
- Две вкладки: **"Актуальные"** и **"Архив"**
- Вкладка "Актуальные" — активна по умолчанию, загружает `getPublicGames()`
- Вкладка "Архив" — загружает `getArchivedGames()` (только для авторизованных)

### 2.2. Логика переключения
- Состояние `activeTab: 'active' | 'archived'`
- При переключении на "Архив" — проверять авторизацию (наличие токена)
- Если неавторизован — показать сообщение "Войдите, чтобы просмотреть архив"

### 2.3. Фильтр по городу
- Оставить фильтр по городу для обеих вкладок
- Для архивных игр — передавать city в `getArchivedGames()`

## 4. Схема работы

```mermaid
flowchart TD
    A[Header] --> B{Пользователь авторизован?}
    B -->|Нет| C[Ссылка \"Каталог игр\" → /games]
    B -->|Да| D[Dropdown \"Каталог игр\"]
    D --> E[\"Актуальные\" → /games]
    D --> F[\"Архив игр\" → /games/archive]

    G[Страница /games] --> H[Вкладки]
    H --> I[\"Актуальные\"]
    H --> J[\"Архив\"]
    I --> K[getPublicGames - PUBLISHED, REGISTRATION_OPEN и др.]
    J --> L{Авторизован?}
    L -->|Да| M[getArchivedGames - только ARCHIVED]
    L -->|Нет| N[Сообщение \"Войдите, чтобы просмотреть архив\"]
```

## 5. Важные замечания

- **Авторизация для архива**: `getArchivedGames()` уже защищён на бэкенде (`JwtAuthGuard`). Если неавторизованный пользователь попробует переключиться на вкладку "Архив", API вернёт 401. Нужно обработать это на фронтенде — показать сообщение, а не ошибку.
- **Страница `/games/archive`**: остаётся как отдельный маршрут, но ссылка на неё теперь только через dropdown в Header. Можно также оставить редирект с `/games/archive` на `/games` с вкладкой "Архив", но это опционально.
- **Фильтр по городу**: для архивных игр API уже поддерживает `city` параметр.
