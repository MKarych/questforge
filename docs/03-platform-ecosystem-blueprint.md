## 1. "Steam для мероприятий" (Метафора)

**Steam** не продает игры. Steam продает **платформу**. Игры делают тысячи разработчиков.

Мы делаем то же самое. Только вместо игр — **мероприятия**.

| Компонент | В Steam | В Adventure Engine |
| :--- | :--- | :--- |
| **Платформа** | Steam | Adventure Engine |
| **Разработчики** | Игровые студии | Авторы сценариев |
| **Продавцы** | Магазины / дистрибьюторы | Организаторы |
| **Покупатели** | Игроки | Игроки (покупают билеты) |
| **Продукт** | Игры | Game Packages (пакеты мероприятий) |
| **Монетизация** | Комиссия с продаж | Комиссия с продаж + подписка |

**Ключевое отличие:** В Steam разработчик создает игру один раз и продает ее бесконечно. В Adventure Engine автор создает сценарий один раз, а организаторы по всему миру покупают лицензии и проводят мероприятия.

## 3. Game Package (Пакет игры)

Мы продаем не "файл с загадками". Мы продаем **готовый бизнес-комплект**.

### 3.1. Состав Game Package

| Компонент | Описание | Пример |
| :--- | :--- | :--- |
| **Сценарий** | Логика, тексты, вопросы, загадки | "Найди код на колонне" |
| **Медиа** | Фото, видео, музыка, презентации | Фото локаций, видео-подсказки |
| **Настройки** | Таймеры, правила, подсчет очков | Таймер 10 минут, штраф 5 очков |
| **Инструкция организатора** | Как проводить игру, шаблоны рекламы | Готовые посты для соцсетей |
| **Оформление** | Постер, баннеры для соцсетей | Постер игры, афиша |

### 6.2. Как работает продажа

1. Автор создает Game Package.
2. Нажимает **"Опубликовать"**.
3. Маркетплейс проверяет сценарий (валидация).
4. Пакет появляется в каталоге.
5. Организатор покупает лицензию.
6. Автор получает роялти.
7. Платформа получает комиссию.
