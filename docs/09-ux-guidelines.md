# UX Guidelines: Полная UX-архитектура платформы

> **Кодовое имя проекта:** Adventure Engine
> **Уровень документа:** UX-архитектура и дизайн-система
> **Статус:** Утвержден

---

## 1. Принципы дизайна

### 1.1. Простота
> **Минимум кликов, максимум понятности.**

- Игрок делает не более 2 действий на экране.
- Организатор находит нужную функцию за 3 клика.
- Никаких лишних полей, вопросов, шагов.

### 1.2. Скорость
> **Страницы загружаются < 1 секунды.**

- Минимальный вес страниц.
- Кэширование статики.
- Оптимизация изображений.

### 1.3. Доверие
> **Пользователь знает, что происходит.**

- Каждое действие имеет визуальный фидбек.
- Ошибки объяснены понятным языком.
- Прогресс всегда виден.

### 1.4. Адаптивность
> **Работает на любом устройстве.**

- Телефон (320px) → Планшет → Десктоп (1920px).
- Все компоненты резиновые.
- Touch-friendly (кнопки > 44px).

### 1.5. Фидбек
> **Каждое действие — реакция.**

- Нажатие кнопки → спиннер или анимация.
- Отправка ответа → успех/ошибка.
- Переход между заданиями → плавная анимация.

## 2. Цветовая схема

| Назначение | HEX | Пример |
| :--- | :--- | :--- |
| Primary (Основной) | `#3B82F6` | Кнопки, ссылки, акценты |
| Background (Фон) | `#0F1117` | Основной фон страницы |
| Surface (Карточки) | `#181B26` | Карточки, панели, инпуты |
| Border (Границы) | `#2A2F3F` | Разделители, рамки |
| Text (Текст) | `#E4E4E7` | Основной текст |
| Text Dim (Второстепенный) | `#888EA8` | Подписи, хелперы |
| Success (Успех) | `#059669` | Успешные действия |
| Error (Ошибка) | `#DC2626` | Ошибки, предупреждения |
| Warning (Предупреждение) | `#F59E0B` | Внимание |
| Pending (Ожидание) | `#F59E0B` | Ожидание проверки |

| **Secondary** | [ Отмена ] | `background: transparent; color: #E4E4E7; border: 1px solid #2A2F3F; padding: 12px 24px; border-radius: 12px;` |
| **Danger** | [ Удалить ] | `background: #DC2626; color: white; padding: 12px 24px; border-radius: 12px;` |
| **Ghost** | [ Подробнее ] | `background: transparent; color: #3B82F6; padding: 8px 16px;` |

## 5. Структура сайта

### 5.1. Уровни доступа

| Уровень | Описание | Доступ |
| :--- | :--- | :--- |
| **Публичный** | Любой пользователь (включая незарегистрированных) | Главная, каталог игр, страница игры, отзывы, новости |
| **Игрок** | Зарегистрированный пользователь | Участие в играх, команды, профиль, история игр |
| **Организатор** | Подтвержденный организатор | Организаторская панель, создание игр, проверка ответов |
| **Администратор** | Сотрудник платформы | Модерация, управление пользователями, статистика |

| `ERR_NO_CONNECTIONS` | Предупреждение | Узлы не соединены |

## 20. UI как реакция на события (Event-Driven Behavior)

### 20.1. Принцип

> **UI обновляется ТОЛЬКО через события.**

UI не запрашивает состояние напрямую. Он реагирует на поток событий от Engine.

### 20.2. События, на которые реагирует UI

| Событие | Действие в UI |
| :--- | :--- |
| `PLAYER_JOINED` | Обновить список команд |
| `PLAYER_LEFT` | Обновить список команд |
| `GAME_STATE_CHANGED` | Изменить баннер состояния |
| `TASK_COMPLETED` | Анимация завершения, переход к следующему |
| `SCORE_UPDATED` | Обновить счет |
| `TIMER_TICK` | Обновить таймер |
| `NODE_ENTER` | Показать новое задание |
| `NODE_EXIT` | Анимация выхода из задания |
| `ERROR_OCCURRED` | Показать ошибку |

### 20.3. Реализация

```typescript
function useGameEvents(sessionId: string) {
  const [state, setState] = useState<GameState>();

  useEffect(() => {
    const socket = connectWebSocket(sessionId);
    
    socket.on('GAME_STATE_CHANGED', (payload) => {
      setState(payload);
    });
    
    socket.on('NODE_ENTER', (payload) => {
      showTask(payload);
    });
    
    socket.on('SCORE_UPDATED', (payload) => {
      animateScore(payload);
    });
    
    return () => socket.disconnect();
  }, [sessionId]);
}
```

## 22. UX деградация (Подсказки и Fallback)

### 22.1. Временная шкала

| Время | Действие |
| :--- | :--- |
| **10 секунд** | Мягкая подсказка (иконка "?" с намеком) |
| **30 секунд** | Явная подсказка ("Попробуйте посмотреть на ...") |
| **60 секунд** | Авто-подсказка (показывается автоматически) |
| **120 секунд** | Предложение пропустить задание (со штрафом) |

### 22.2. Реализация

```typescript
class HintSystem {
  private timer: NodeJS.Timeout;
  private hintLevel: 0 | 1 | 2 | 3 = 0;

  start(node: Node) {
    this.hintLevel = 0;
    this.timer = setInterval(() => {
      this.hintLevel++;
      this.showHint(this.hintLevel, node);
    }, 30000);
  }

  showHint(level: number, node: Node) {
    switch (level) {
      case 1:
        this.showSoftHint(node.hintSoft);
        break;
      case 2:
        this.showExplicitHint(node.hint);
        break;
      case 3:
        this.showSkipOption(node);
        break;
    }
  }
}
```

## 19. Итоговые принципы

1. **Минимализм.** Ничего лишнего. Только то, что нужно пользователю.
2. **Скорость.** Страницы загружаются моментально, действия откликаются мгновенно.
3. **Понятность.** Пользователь всегда знает, что происходит и что делать дальше.
4. **Доверие.** Интерфейс выглядит профессионально и надежно.
5. **Доступность.** Интерфейс работает для всех пользователей.
6. **Event-Driven.** UI реагирует на события, а не на запросы.
7. **Offline-First.** Интерфейс работает без интернета.
8. **State-Aware.** UI всегда отражает текущее состояние системы.
