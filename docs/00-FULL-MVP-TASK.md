## 1. ОБЩАЯ ЦЕЛЬ

Сделать работающий MVP платформы "Город Приключений", чтобы:
- Организатор мог **создать игру** и **сценарий**
- Игрок мог **найти игру в каталоге** и **пройти её**
- Всё работало **без багов** и **без красивой картинки**

---

## 2. ЧТО ДОЛЖНО РАБОТАТЬ (ЧЕКЛИСТ)

### 2.1. Публичная часть (без авторизации)
- [ ] Главная страница показывает **реальные игры** из БД
- [ ] Каталог игр (`/games`) показывает **все опубликованные игры**
- [ ] Страница игры (`/games/[id]`) показывает **описание, дату, город, организатора**
- [ ] Отзывы на игры **сохраняются и отображаются**

### 2.2. Аутентификация
- [ ] Регистрация работает **и создаёт пользователя в БД**
- [ ] Логин работает **и выдаёт JWT токен**
- [ ] Токен **сохраняется в localStorage**
- [ ] Выход **очищает токен**

### 2.3. Организатор
- [ ] Заявка на организатора **сохраняется в БД**
- [ ] Статус заявки **отображается пользователю**
- [ ] Модератор может **одобрить/отклонить заявку**

### 2.4. Создание игры
- [ ] Организатор может **создать игру** (название, город, дата, цена)
- [ ] Игра **сохраняется в БД**
- [ ] Игра появляется в **списке игр организатора**
- [ ] Игру можно **отправить на модерацию**
- [ ] Модератор может **одобрить/отклонить игру**
- [ ] Одобренная игра **появляется в публичном каталоге**

### 2.5. Сценарии
- [ ] Организатор может **создать сценарий** (в конструкторе)
- [ ] Сценарий **сохраняется в БД**
- [ ] Сценарий **валидируется** (есть START, FINISH, нет циклов)
- [ ] Сценарий можно **опубликовать**

### 2.6. Игрок
- [ ] Игрок может **зайти по ссылке** на игру
- [ ] Игрок может **ввести название команды**
- [ ] Игрок видит **текущее задание**
- [ ] Игрок может **отправить ответ** (текст, код, фото)
- [ ] Игрок получает **фидбек** (правильно/неправильно)
- [ ] Игрок видит **финиш** (счёт, время)

### 2.7. Админ
- [ ] Админ видит **список игр на модерации**
- [ ] Админ может **одобрить/отклонить игру**
- [ ] Админ видит **список заявок организаторов**
- [ ] Админ может **одобрить/отклонить заявку**

#### 9.3. Технические требования

| Что | Как |
|-----|-----|
| **Аватар** | Загружается в MinIO / S3, возвращается URL, сохраняется в `users.avatarUrl` |
| **Данные профиля** | `PATCH /users/me` — обновление имени, города, био, контактов |
| **Приватность** | `PATCH /users/me/privacy` — настройки видимости |
| **Статистика** | Считается на лету: `COUNT(games)`, `COUNT(sessions)`, `COUNT(scenarios)` |
| **Достижения** | Хранятся в `users.achievements` (JSON-массив) |
| **Рейтинг** | Средняя оценка отзывов, где пользователь — организатор или автор |

---

#### 9.4. Эндпоинты

| Метод | URL | Описание |
|-------|-----|----------|
| `GET` | `/users/:id` | Публичный профиль |
| `GET` | `/users/me` | Мой профиль (с email, настройками) |
| `PATCH` | `/users/me` | Обновить профиль |
| `POST` | `/users/me/avatar` | Загрузить аватар |
| `GET` | `/users/me/statistics` | Моя статистика (для дашборда) |
