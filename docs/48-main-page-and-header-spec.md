> **Дата:** 24.06.2026  
> **Статус:** Утвержден  
> **Версия:** 3.0  
> **Класс:** Архитектурный контракт (10/10 — Enterprise Ready)  
> **Цель:** Описать главную страницу и шапку как живую, адаптивную, масштабируемую и производительную систему, готовую к росту до 1M+ пользователей.

---

## 1. Архитектурные принципы

- **Главная страница — точка входа в платформу.** Она должна отвечать на три вопроса за 3 секунды.
- **Шапка (Header) — навигационный центр и командный центр.** 
- **Агрегированный DTO.** Все данные для главной страницы получаются одним запросом (`GET /home`).
- **Компонентный подход.** Каждый блок — независимый компонент со своей логикой, состояниями и контрактом.
- **Feature Flags управляют видимостью.** Модули включаются/выключаются через конфиг.
- **SSR/ISR/CSR стратегия.** Каждый блок рендерится оптимальным способом.
- **Производительность и SEO — обязательны.**
- **Права доступа определяют видимость.**

5. Состояния и Error Boundaries
5.1. Состояния страниц
Загрузка (Loading): Скелетоны для всех блоков (приоритет: Hero → Games → Reviews → Teams → FAQ)

Успех (Success): Все данные загружены

Частичная загрузка: Некоторые блоки загружены, другие показывают ошибку

Пустая (Empty): Нет игр, команд, организаторов → показывать заглушки

Ошибка (Error): "Не удалось загрузить страницу" + кнопка "Повторить"

5.2. Error Boundary
Каждый блок (Games, Reviews, Teams, FAQ, Organizers) обёрнут в ErrorBoundary

Если один блок упал, страница не падает целиком

Пользователь видит: ⚠️ Не удалось загрузить блок. [Повторить]

5.3. Empty State
Иконка: 🎮 (для игр), 👥 (для команд), ⭐ (для отзывов)

Текст: "Пока пусто. Будьте первыми!"

CTA: Кнопка "Создать игру" / "Стать организатором"

5.4. Retry Strategy
Стратегия: 3 автоматических попытки (с экспоненциальной задержкой)

Если не удалось: показать кнопку "Повторить" с ручным retry

5.5. Offline Mode
Индикатор: SystemStatusBar показывает 🔴 Offline

Поведение: показывать кэшированные данные (если есть), иначе заглушку

15. Архитектурные правила (Контракт для агентов)
Главная страница — Read-Only. Она не отправляет данные на сервер (кроме поиска и аналитики).

Все данные через единый API (GET /home). Никаких отдельных запросов к /games, /teams, /reviews.

Компоненты не знают о бизнес-логике. Только получают данные через пропсы.

Стейт-менеджмент: только локальный (useState) или через React Query.

SEO обязателен. Все мета-теги должны быть заполнены.

Error Boundary — обязателен для каждого блока.

Feature Flags управляют видимостью блоков. Если модуль выключен — блок не показывается.

Skeleton обязателен для всех асинхронных блоков.

Empty State обязателен для всех пустых блоков.

Адаптивность обязательна.

Доступность (Accessibility) обязательна.
