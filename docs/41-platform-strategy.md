# Стратегия платформы: Как построить Game Runtime Platform

> **Кодовое имя проекта:** Город Приключений (Adventure City)
> **Цель:** Стать универсальной платформой для любых игр и мероприятий.
> **Принцип:** Любая механика конкурентов должна быть реализуема нашим конструктором без кода.
> **Дата:** 23.06.2026

---

## 1. Философия платформы

Мы строим не "очередной сайт с квестами" и не "убийцу Encounter". Мы строим **Game Runtime Platform** — среду, в которой можно создать любую игру, не написав ни строчки кода.

**Encounter, Geocaching, квизы, корпоративные тимбилдинги, музейные квесты — это всё просто пресеты (готовые шаблоны) для нашей платформы.**

---

## 2. Архитектурные слои

Чтобы достичь такой гибкости, мы строим систему из трёх слоёв:

### 2.1. Ядро (Core Engine)
- **Универсальные примитивы:** Текст, Код, GPS, QR, Таймер, Фото, Видео, Диалог, Предмет и т.д.
- **Event Sourcing:** Все действия игроков — это события, из которых можно пересобрать любое состояние.
- **Гибкая логика:** Ветвления, условия, переменные, циклы, рандом.

**Статус:** ✅ В разработке (есть основа)

### 2.2. Конструктор (Builder)
- **Визуальный редактор:** На основе React Flow (уже есть!)
- **Палитра примитивов:** Автор просто перетаскивает блоки и настраивает их.
- **Сборка игр:** Из примитивов собирается любой сценарий.

**Статус:** ✅ Базовая версия готова

### 2.3. Платформа (Platform)
- **Интерфейсы:** Для игроков, организаторов, авторов, судей, наблюдателей.
- **Сообщество:** Профили, рейтинги, форумы, команды.
- **Экономика:** Платежи, маркетплейс, подписки, реклама.

**Статус:** ⏳ В разработке (есть основа)

## 4. Наша стратегия превосходства

### 4.1. Что мы делаем лучше

| Критерий | Конкуренты | Мы |
| :--- | :--- | :--- |
| **Гибкость** | Жёсткие форматы (игра только "Схватка" или "Квиз") | **Один универсальный движок** для всех форматов. |
| **Конструктор** | Текстовый, сложный, или его нет | **Визуальный редактор (React Flow)** — собирай как Lego. |
| **AI** | Нет или очень ограничен | **AI-генерация сценариев, AI-проверка фото, AI-помощник.** |
| **Экономика** | Закрытая (взносы) | **Открытый маркетплейс** сценариев, плагинов и шаблонов. |
| **Дизайн** | Устаревший (1990-2010) | **Современный, тёмный, адаптивный.** |
| **Целевая аудитория** | Городские квесты | **Городские квесты + Квизы + Образование + Бизнес + Развлечения.** |
| **Технологии** | Легаси-код (ASP.NET WebForms) | **Современный стек (NestJS, Next.js).** |

### 4.2. Наша "философия победы"

> **"Любая игра, которую можно придумать, должна собираться на нашей платформе без программирования. Весь фокус — на универсальность и простоту."**

---

## 5. Ключевое отличие (УТП)

1. **Универсальность:** Наша платформа — это не набор игр, а **операционная система для мероприятий**.
2. **Простота:** Любой человек, даже без опыта, может создать свою игру.
3. **Экосистема:** Авторы зарабатывают, организаторы проводят, игроки развлекаются. Все в выигрыше.

## 6. Дорожная карта на год

### 6.1. Ближайший релиз (v1.0)
- ✅ **Завершить MVP:** Регистрация, игры, команды, публичная страница игры, Лобби.
- ⏳ **Запустить в бета-тест** с реальными организаторами (июль 2026).

### 6.2. Развитие (v2.0) — 3-6 месяцев
1. Добавить универсальные механики: квизы, голосование, систему предметов, агентов (NPC).
2. Сделать маркетплейс сценариев и платежи.
3. Интегрировать AI-генерацию сценариев.

### 6.3. Масштабирование (v3.0+) — 6-12 месяцев
1. Выйти за рамки городских игр: добавить поддержку образовательных, бизнес- и музейных форматов.
2. Сделать SDK и API для разработчиков, чтобы они могли создавать свои плагины и механики.
3. Запустить глобальную сеть доменов (по городам и странам).

## 8. Итог

Мы строим **Game Runtime Platform**, которая сделает все существующие решения для игр и мероприятий устаревшими.

**Encounter — лишь один из шаблонов, который можно собрать на нашей платформе за считанные минуты.**

Наша цель — не заменить конкурентов, а предложить рынку инструмент, который сделает их формат лишь частью гораздо большей экосистемы.
