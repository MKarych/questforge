# Game Mechanics Specification: Полная классификация механик платформы

> **Кодовое имя проекта:** Adventure Engine
> **Уровень документа:** Архитектурный
> **Статус:** Утвержден

---

## 1. Принципы классификации

Все механики разделены по функциональным слоям:

| Слой | Описание |
| :--- | :--- |
| **Mission Mechanics** | То, что игрок выполняет (задания) |
| **Transition Mechanics** | Как система решает, куда идти дальше |
| **Time Mechanics** | Управление временем в игре |
| **Hint Mechanics** | Система подсказок |
| **Inventory Mechanics** | Предметы и инвентарь |
| **Resource Mechanics** | Параметры команды |
| **Role Mechanics** | Роли игроков |
| **PvE Mechanics** | Игрок против мира |
| **PvP Mechanics** | Игрок против игрока |
| **Social Mechanics** | Чат, союзы, обмен |
| **World Mechanics** | Карта, территории, фракции |
| **AI Mechanics** | Искусственный интеллект |
| **Organizer Mechanics** | Инструменты организатора |
| **Game Modes** | Режимы игры (жанры) |

## 2. Mission Mechanics (Механики заданий)

### Базовые (MVP)

| # | Механика | Описание | Валидация |
| :--- | :--- | :--- | :--- |
| 1 | **Текстовый ответ** | Игрок вводит текст | Точное совпадение / Regex |
| 2 | **Кодовый ответ** | Игрок вводит цифровой/буквенный код | Точное совпадение |
| 3 | **Фотоотчет** | Игрок загружает фото | Ручная / AI проверка |
| 4 | **Видеоотчет** | Игрок загружает видео | Ручная проверка |
| 5 | **GPS-точка** | Игрок должен прийти в координаты | Проверка радиуса |
| 6 | **QR-код** | Игрок сканирует QR | Точное совпадение |
| 7 | **Выбор варианта** | Игрок выбирает из списка | Совпадение с правильным |
| 8 | **Таймер** | Ожидание или дедлайн | Автоматический переход |
| 9 | **Последовательность** | Действия в строгом порядке | Проверка порядка |

### Продвинутые (v1.0+)

| # | Механика | Описание | Валидация |
| :--- | :--- | :--- | :--- |

| 11 | **Голосовой ответ** | Игрок отвечает голосом | Распознавание речи |
| 12 | **NFC-метка** | Прикосновение к метке | Проверка сигнала |

## 3. Transition Mechanics (Механики переходов)

| # | Тип перехода | Описание | Пример |
| :--- | :--- | :--- | :--- |
| 1 | **Линейный** | Всегда на следующий узел | A → B → C |
| 2 | **Условный (success/fail)** | По результату задания | Код верный → B, неверный → C |
| 3 | **Случайный** | С вероятностью | 50% → B, 50% → C |
| 4 | **По инвентарю** | По наличию предмета | Есть ключ → открыть дверь |
| 5 | **По очкам** | По количеству очков | Очки > 100 → финал |
| 6 | **По времени** | По времени суток | Ночь → сценарий N, День → сценарий D |
| 7 | **По условию (custom)** | Пользовательское условие | Если score > 50 AND hasItem('ключ') |

## 4. Time Mechanics (Механики времени)

| # | Механика | Описание |
| :--- | :--- | :--- |
| 1 | **Общий таймер игры** | Лимит времени на всю игру |
| 2 | **Таймер этапа** | Лимит времени на конкретное задание |
| 3 | **Отложенное событие** | Событие через N минут после старта |
| 4 | **Расписание** | Событие в конкретное время |
| 5 | **Time Travel** | Откат состояния игры к предыдущему моменту |
