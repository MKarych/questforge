## 2. Верхняя панель (логика расположения)

> **Слева — навигация и идентификация (что я редактирую). Справа — действия (что я делаю).**

### Левая часть (навигация и идентификация)

| Элемент | Поведение |
| :--- | :--- |
| **← Назад** | Выход с подтверждением "Вы уверены? Несохранённые изменения будут потеряны". Ведёт на `/organizer/scenarios`. |
| **Название сценария** | Редактируется по клику (инлайн-редактор). Сохраняется вместе со сценарием. |
| **Статус сценария** | Индикатор: `DRAFT` (серый), `PUBLISHED` (зелёный). Показывается рядом с названием. |

### Правая часть (действия)

| Элемент | Поведение |
| :--- | :--- |
| **💾 Сохранить** | Сохраняет сценарий. Показывает тост "✅ Сценарий сохранён" или "❌ Ошибка: ..." |
| **👁 Превью** | Открывает модальное окно с имитацией телефона. Показывает текущий узел, таймер, ответ. |
| **🎮 Тест** | Открывает модальное окно, где автор проходит сценарий как игрок. Показывает счёт, переменные, лог. |
| **📤 Опубликовать** | Меняет статус сценария на `PUBLISHED`. Показывает тост "✅ Сценарий опубликован". |

### Поведение блоков

| Действие | Реакция |
| :--- | :--- |
| **Перетаскивание блока** | Блок появляется на холсте (в центре или рядом с выбранным) |
| **Наведение на блок** | Подсветка блока в палитре |
| **Клик на блок (если не перетаскивать)** | Создаёт новый блок на холсте (в центре видимой области) |

### Состояния блоков в палитре

| Состояние | Визуал |
| :--- | :--- |

| **Наведение** | Подсветка синим (`#3B82F6`) |

## 4. Холст (React Flow)

### Поведение

| Действие | Реакция |
| :--- | :--- |
| **Перетаскивание блока** | Блок перемещается. Сохраняется в истории (Undo/Redo). |
| **Клик на блок** | Блок выделяется (синяя обводка). Открываются настройки справа. |
| **Клик на пустое место** | Снимается выделение. Настройки скрываются. |
| **Соединение блоков** | Перетащить из порта в порт. Создаётся ребро. |
| **Клик на ребро** | Ребро выделяется (оранжевое). Можно удалить по Delete. |
| **Наведение на ребро** | Ребро подсвечивается. Появляется кнопка "✕". |

### Визуальные состояния блоков

| Состояние | Визуал |
| :--- | :--- |
| **Обычный** | Тёмный фон, белый текст, иконка слева |
| **Выделенный** | Синяя обводка (`#3B82F6`) |
| **С ошибкой** | Красная обводка (`#DC2626`) + иконка ошибки |
| **Заблокированный** | Серый, нельзя редактировать (только START и FINISH) |

## 6. Горячие клавиши

| Клавиша | Действие |
| :--- | :--- |
| `Delete` / `Backspace` | Удалить выделенный блок ИЛИ выделенное ребро |
| `Esc` | Снять выделение со всего |
| `Ctrl+C` (Mac: `Cmd+C`) | Копировать выделенный блок |
| `Ctrl+V` (Mac: `Cmd+V`) | Вставить скопированный блок (+50px offset) |
| `Ctrl+Z` (Mac: `Cmd+Z`) | Отменить последнее действие |
| `Ctrl+Shift+Z` (Mac: `Cmd+Shift+Z`) | Повторить отменённое действие |
| `Ctrl+S` (Mac: `Cmd+S`) | Сохранить сценарий |
| `Ctrl+A` (Mac: `Cmd+A`) | Выделить все блоки |

## 10. Валидация

| Проверка | Когда | Ошибка |
| :--- | :--- | :--- |
| **Есть START** | При сохранении | "❌ Сценарий должен иметь START-блок" |
| **Есть FINISH** | При сохранении | "❌ Сценарий должен иметь FINISH-блок" |
| **Все узлы достижимы** | При сохранении | "❌ Узел [название] недостижим" |
| **Нет циклов** | При сохранении | "❌ Обнаружен бесконечный цикл" |
| **Заполнены обязательные поля** | При сохранении | "❌ У блока [название] не заполнен ответ" |

## 11. Итоговые требования к UX

1. **Все изменения сохраняются мгновенно** (по Ctrl+S или через автосохранение).
2. **Интерфейс адаптивен** — работает на экранах от 1024px и выше.
3. **Все ошибки понятны пользователю** — без "Internal Server Error".
4. **Интерактивность** — перетаскивание, выделение, подсветка, фидбек на каждое действие.
5. **Горячие клавиши** — работают на Windows и Mac.
6. **AR-блок** — виден, но не активен (заглушка).

## 12. Удаление START и FINISH

- START и FINISH можно удалять как обычные блоки (через Delete или кнопку "Удалить блок").
- После удаления START или FINISH сценарий становится невалидным.
- При попытке сохранить невалидный сценарий — ошибка "❌ Сценарий должен иметь START и FINISH".
- Автор должен добавить новый START и/или FINISH, чтобы сценарий стал валидным.
