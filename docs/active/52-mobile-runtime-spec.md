---

## 1. Принципы

1. **Offline-first.** Игрок может проходить игру без интернета.
2. **Отложенная синхронизация.** Все изменения синхронизируются при восстановлении соединения.
3. **Локальное хранение.** Все данные кэшируются на устройстве.
4. **Конфликты.** Система умеет разрешать конфликты при синхронизации (сервер имеет приоритет).
5. **Локальная + серверная проверка GPS.** Локальная предварительная проверка + серверная финальная.

Все медиа (фото, аудио, видео) загружаются заранее

text

3. Синхронизация
3.1. Очередь событий
typescript
interface PendingEvent {
  id: string;
  type: 'answer' | 'photo' | 'gps' | 'hint' | 'sos';
  payload: any;
  timestamp: Date;
  retries: number;
}
3.2. Стратегия синхронизации
text
1. Проверить соединение
2. Если есть → отправить все pending events
3. Получить обновленное состояние от сервера
4. Обновить локальное состояние
5. Очистить отправленные события
3.3. Конфликты
text
Если сервер вернул состояние, которое конфликтует с локальным:
→ Сервер всегда имеет приоритет
→ Локальные изменения перезаписываются

5. GPS и камера
5.1. GPS
text
GPS координаты запрашиваются при входе в сцену
Локальная предварительная проверка (на устройстве)
Серверная финальная проверка (при синхронизации)
Если проверка не совпадает → ошибка "GPS не подтверждён"
Максимальная скорость перемещения между точками (пешком 5 км/ч, авто 60 км/ч)
5.2. Камера
text
Фото сохраняются локально
Отправляются при синхронизации
Поддерживается загрузка нескольких фото
