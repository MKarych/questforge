## 1. Принципы роста

### 1.1. Сетевой эффект с тремя контурами

У нас не одна аудитория, а **три** (игроки, авторы, организаторы), и каждая усиливает другие.

```text
Больше авторов → Больше игр → Больше игроков
                     ↑              ↓
    Авторы зарабатывают ← Больше организаторов ← Игроки платят за билеты
```

### 1.2. Лестница роста

Мы не ищем "готовых" профессионалов. Мы **выращиваем** их внутри платформы.

```text
Посетитель → Игрок → Активный игрок → Автор-Организатор → Профи → Чистый Автор / Чистый Организатор
```

### 1.3. Публичная часть как воронка

Платформа имеет **публичную часть**, которая работает как воронка для новых пользователей:

```text
Посетитель (увидел анонс)
    ↓
Заинтересовался (посмотрел каталог, страницу игры)
    ↓
Зарегистрировался (стал игроком)
    ↓
Сыграл в игру (получил эмоции)
    ↓
Захотел создать свою игру (стал автором-организатором)
```

### 1.4. Сообщество как якорь

Инструменты можно скопировать. Сообщество — нельзя.

### 1.5. Платформа как активный участник

Мы не ждем, пока пользователи найдут друг друга. Мы:
- Продвигаем лучшие игры
- Организуем конкурсы
- Создаем спрос (реклама)
- Помогаем авторам зарабатывать

## 2. Публичная часть (Воронка новых пользователей)

### 2.1. Что видят все пользователи (без регистрации)

| Раздел | Описание | Цель |
| :--- | :--- | :--- |
| **Главная** | Анонсы игр, ближайшие события, топ-игр | Заинтересовать, показать ценность |
| **Каталог игр** | Список всех игр с фильтрами (город, дата, тип) | Помочь найти игру |
| **Страница игры** | Описание, дата, город, отзывы, обсуждения | Убедить зарегистрироваться и участвовать |
| **Новости** | Новости платформы, интервью, гайды | Создать доверие и экспертность |
| **Для организаторов** | Как стать организатором, что это дает | Привлечь новых организаторов |

### 2.2. Путь пользователя к регистрации

```text
1. Пользователь заходит на сайт
   ↓
2. Видит ближайшие игры в своем городе
   ↓
3. Переходит на страницу игры
   ↓
4. Читает описание, смотрит отзывы
   ↓
5. Нажимает "Участвовать"
   ↓
6. Видит: "Для участия необходимо зарегистрироваться"
   ↓
7. Регистрируется
   ↓
8. Становится игроком
```

## 3. Роли пользователей (расширенная модель)

### 3.1. Пять ролей

| Роль | Суть | Что делает | Что получает |
| :--- | :--- | :--- | :--- |
| **Посетитель** | Незарегистрированный пользователь | Смотрит каталог, читает отзывы | Информация об играх |
| **Игрок** | Зарегистрированный пользователь | Участвует в играх, создает команды | Эмоции, досуг, рейтинг |
| **Автор-Организатор** | **Создает и проводит** | Пишет сценарий → проводит → продает (опционально) | Зарабатывает, прокачивает навыки |
| **Чистый Автор** | Креативный специалист | Пишет сценарии, продает на маркетплейсе | Пассивный доход |
| **Чистый Организатор** | Предприниматель | Покупает готовые пакеты, проводит игры | Готовый бизнес |

### 3.2. Ключевая роль: Автор-Организатор

Это наше **ядро**. Именно он:
- Тестирует механики в бою
- Улучшает сценарии на основе фидбека
- Затем выходит на маркетплейс с готовым продуктом
- Становится примером для других

## 4. Лестница роста (Подробно)

### Шаг 1: Посетитель

**Как приходит:** реклама, сарафанное радио, соцсети, поиск.

**Что делает:**
- Смотрит каталог игр
- Читает отзывы
- Изучает страницы игр

**Что мы даем:**
- Красивый, понятный сайт
- Актуальные анонсы игр
- Отзывы и рейтинги

**Наша цель:** превратить в зарегистрированного игрока.

---

### Шаг 2: Игрок

**Как приходит:** зарегистрировался, чтобы участвовать в игре.

**Что делает:**
- Участвует в 1-2 играх
- Оставляет отзывы
- Создает команду с друзьями

**Что мы даем:**
- Легкий вход (по ссылке)
- Качественный игровой опыт
- Красивый интерфейс

**Наша цель:** превратить в активного игрока.

---

### Шаг 3: Активный игрок

**Как приходит:** прошел 3+ игры, ему понравилось.

**Что делает:**
- Проходит 5+ игр в месяц
- Ищет новые игры в каталоге
- Оставляет развернутые отзывы
- Приглашает друзей

**Что мы даем:**
- Подписка на любимых авторов
- Календарь игр
- Рейтинг игрока
- Сезонные события

**Наша цель:** показать, что он может сделать свою игру.

---

### Шаг 4: Автор-Организатор (Новичок)

**Как приходит:** открыл конструктор впервые.

**Что делает:**
- Создает простую игру для 5-10 друзей
- Проводит ее
- Получает фидбек

**Что мы даем:**
- Интуитивный конструктор
- Пресеты (готовые шаблоны)
- Подсказки и обучение
- Инструкции по проведению

**Наша цель:** провести первую платную игру.

---

### Шаг 5: Автор-Организатор (Профи)

**Как приходит:** провел 3+ платных игры.

**Что делает:**
- Проводит 1-2 игры в месяц
- Зарабатывает деньги
- Улучшает свои сценарии
- Появляется репутация

**Что мы даем:**
- Прием платежей
- Аналитику игры
- Помощь в рекламе

**Наша цель:** масштабироваться до 10 игр в месяц.

---

### Шаг 6a: Чистый Автор

**Как приходит:** понял, что организатор — не его путь.

**Что делает:**
- Пишет сценарии
- Выкладывает на маркетплейс
- Получает роялти

**Что мы даем:**
- Маркетплейс с аудиторией
- Инструменты для создания Game Package
- Защиту от копирования

**Наша цель:** стабильный пассивный доход.

---

### Шаг 6b: Чистый Организатор

**Как приходит:** понял, что писать сценарии — не его путь.

**Что делает:**
- Покупает готовые Game Packages
- Проводит игры
- Масштабирует бизнес

**Что мы даем:**
- Каталог сценариев
- Инструменты для проведения
- Помощь в рекламе

**Наша цель:** проводить 10+ игр в месяц.

## 5. Онбординг организаторов

### 5.1. Как стать организатором

| Шаг | Что происходит | Где |
| :--- | :--- | :--- |
| **1** | Пользователь регистрируется на сайте | `/auth/register` |
| **2** | Заполняет профиль (имя, город, контакты) | `/settings/profile` |
| **3** | Подает заявку на статус организатора | `/organizer/apply` |
| **4** | Модератор проверяет заявку | Админ-панель (1-3 дня) |
| **5** | Получает доступ к организаторской панели | Email-уведомление |

### 5.2. Что проверяется при модерации

| Проверка | Описание |
| :--- | :--- |
| **Контакты** | Есть телефон, Telegram/WhatsApp |
| **Опыт** | Был ли опыт проведения игр (опционально) |
| **Город** | Реальный город, где будет проводить игры |
| **Согласие с правилами** | Публичная оферта, правила платформы |

### 5.3. Статусы заявки

| Статус | Описание | Что видит пользователь |
| :--- | :--- | :--- |
| **PENDING** | Заявка на рассмотрении | "Ваша заявка на рассмотрении" |
| **APPROVED** | Одобрено | "Поздравляем! Вы теперь организатор" |
| **REJECTED** | Отклонено | "К сожалению, ваша заявка отклонена. Причина: ..." |

## 6. Модерация игр

### 6.1. Жизненный цикл игры

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. ЧЕРНОВИК (DRAFT)                                           │
│    → Организатор создает игру, заполняет все поля             │
│    → Видит только организатор                                 │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. ОТПРАВЛЕНА НА МОДЕРАЦИЮ (PENDING)                         │
│    → Организатор нажимает "Опубликовать"                      │
│    → Игра попадает в очередь модерации                        │
│    → Видит только организатор и модератор                     │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. ОДОБРЕНА (APPROVED)                                        │
│    → Модератор проверяет: описание, город, дата, контакты     │
│    → Игра появляется в публичном каталоге                     │
│    → Видят все пользователи                                   │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. АКТИВНА (ACTIVE)                                           │
│    → Игра началась (организатор нажал "Старт")                │
│    → Игроки проходят задания                                  │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 5. ЗАВЕРШЕНА (FINISHED)                                       │
│    → Игра завершена, результаты опубликованы                  │
│    → Доступны отзывы и обсуждения                             │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2. Что проверяется при модерации игры

| Проверка | Описание |
| :--- | :--- |
| **Название и описание** | Не содержит оскорблений, рекламы, ссылок |
| **Город** | Корректный город |
| **Дата и время** | Реальная дата (не прошлая) |
| **Контакты** | Есть контакт для связи с организатором |
| **Сценарий** | Прошел валидацию (не сломает Engine) |
| **Цена** | Корректная цена билета (если платная) |

## 8. Качество контента

### 8.1. Модерация сценариев (до публикации)
- Автоматическая валидация (см. `12-scenario-validation-spec.md`)
- Ручная модерация для новых авторов
- Проверка на: дубликаты, вредоносный контент, ошибки

### 8.2. Система рейтинга (после публикации)

| Источник | Вес |
| :--- | :--- |
| Оценки игроков | 50% |
| Количество проведений | 30% |
| Оценки организаторов | 20% |

### 8.3. Работа с негативом
- Жалобы на сценарий (кнопка "Пожаловаться")
- Модерация в течение 24 часов
- Блокировка автора при нарушениях

## 9. Сетевой эффект (Подробно)

Три стороны, которые усиливают друг друга:

| Сторона | Что дает | Что получает |
| :--- | :--- | :--- |
| **Игроки** | Спрос, трафик, отзывы | Досуг, эмоции |
| **Авторы** | Контент, разнообразие игр | Пассивный доход, реализация |
| **Организаторы** | Проведение игр, аудитория | Прибыль, готовый бизнес |
| **Платформа** | Инфраструктура, сообщество | Комиссии, подписки |

**Цикл роста:**
1. Мы привлекаем игроков (реклама, публичная часть).
2. Игроки видят игры авторов.
3. Авторы видят, что спрос есть → пишут больше игр.
4. Организаторы видят спрос → покупают сценарии.
5. Авторы получают деньги → мотивация расти.
6. Игроки видят больше игр → приходят чаще.

## 12. Риски роста и их митигация

| Риск | Вероятность | Решение |
| :--- | :--- | :--- |
| **Снижение качества игр** | Средняя | Система рейтинга, модерация |

| **Сложность роста** | Средняя | Лестница роста, обучение |

| **Недостаток игроков** | Высокая | Публичная часть, реклама, сарафанное радио |
| **Отток организаторов** | Средняя | Помощь в рекламе, качественные сценарии |
| **Сложность онбординга** | Средняя | Понятные инструкции, поддержка |
