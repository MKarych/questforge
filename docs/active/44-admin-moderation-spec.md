# 44. Администрирование и Модерация (Полная спецификация)

> **Кодовое имя:** Город Приключений (Adventure Engine)
> **Уровень:** Продуктовый / Инженерный
> **Статус:** Утвержден. К выполнению.
> **Цель:** Дать четкое разделение ролей `ADMIN` и `MODERATOR` и описать их возможности.

---

## 1. Введение и Роли

Мы внедряем четкую двухуровневую систему управления контентом.

### 1.1. Иерархия ролей

| Роль | Суть | Доступ |
| :--- | :--- | :--- |
| **ADMIN (Бог)** | Полный контроль над платформой. | Всё видит, всё может. Создает модераторов, блокирует пользователей, управляет деньгами. |
| **MODERATOR (Шериф)** | Следит за порядком, чистит контент. | Модерация игр, сценариев и заявок организаторов. Не имеет доступа к деньгам и управлению пользователями. |
| **ORGANIZER (Бизнесмен)** | Основной пользователь платформы. | Создает игры и сценарии. |
| **PLAYER (Народ)** | Потребляет контент. | Участвует в играх. |

### 1.2. Выдача прав (Как стать Админом/Модератором)

*   **Способ:** Только через **Prisma Studio** (прямое редактирование БД).
*   **Действие:** В таблице `users` для нужного пользователя установить поле `role` = `ADMIN` или `MODERATOR`.
*   **Важно:** После смены роли пользователю нужно **выйти и зайти заново** (перевыпустить JWT токен), чтобы система подхватила новые права.

### 3.2. Модерация игр (`/admin/games/pending`)
*Самый важный раздел. Здесь решается, какая игра пойдет в люди.*

**Функционал:**
1.  **Список:** Отображаются игры со статусом `PENDING`.
2.  **Карточка игры:** Название, Город, Организатор, Дата подачи.
3.  **Просмотр:** При клике на название открывается **превью** игры (страница игры в режиме "только чтение").
4.  **Действия:**
    *   **Кнопка "✅ Одобрить"**:
        *   Меняет статус игры на `APPROVED`.
        *   Игра появляется в публичном каталоге `/games`.
        *   Организатор получает уведомление (тост/почта).
    *   **Кнопка "❌ Отклонить"**:
        *   Открывает модальное окно с полем **"Причина отклонения"** (обязательно!).
        *   Меняет статус игры на `REJECTED`.
        *   Организатор видит причину в личном кабинете.
5.  **Безопасность:**
    *   Нельзя удалить игру (только отклонить).
    *   Нельзя редактировать игру.

---

### 3.3. Управление пользователями (`/admin/users`) — ТОЛЬКО ДЛЯ ADMIN
*Инструмент для поддержания дисциплины.*

**Функционал:**
1.  **Список:** Таблица со всеми пользователями (ID, Email, Имя, Роль, Статус).
2.  **Поиск:** Поиск по Email или Имени.
3.  **Действия:**
    *   **Заблокировать:** `status` → `BANNED`. Пользователь не может войти в систему.
    *   **Разблокировать:** `status` → `ACTIVE`.
    *   **Назначить модератором:** Сменить роль с `ORGANIZER` на `MODERATOR` (и наоборот).
4.  **Ограничения:**
    *   **НЕЛЬЗЯ** удалять пользователей (только блокировка).
    *   **НЕЛЬЗЯ** менять пароль (только через механизм восстановления).

---

### 3.4. Модерация заявок (`/admin/organizers/applications`)
*Как обычные люди становятся Организаторами.*

**Функционал:**
1.  **Список:** Заявки со статусом `PENDING`.
2.  **Карточка заявки:** ФИО, Город, Телефон, Telegram, Опыт.
3.  **Действия:**
    *   **"✅ Одобрить"**:
        *   Меняет статус заявки на `APPROVED`.
        *   Меняет роль пользователя на `ORGANIZER`.
        *   Пользователь получает доступ к созданию игр.
    *   **"❌ Отклонить"**:
        *   Открывает поле **"Причина"** (обязательно!).
        *   Меняет статус заявки на `REJECTED`.
        *   Пользователь остается `PLAYER`.

## 4. Техническая реализация (Бэкенд)

### 4.1. Защита эндпоинтов (NestJS Guards)

Все эндпоинты в папке `admin` должны быть жестко защищены.

```typescript
// Пример guards
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles('ADMIN', 'MODERATOR') // Для модерации игр
@Controller('admin/games')
export class AdminGamesController {
  // ...
}

// Только для ADMIN
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles('ADMIN')
@Controller('admin/users')
export class AdminUsersController {
  // ...
}

6. Резюме
Созданы страницы: /admin/dashboard, /admin/games/pending, /admin/games, /admin/organizers/applications, /admin/users.

На всех страницах стоит проверка роли (редирект на / если не ADMIN/MODERATOR).

Кнопки "Одобрить/Отклонить" меняют статус в БД.

При отклонении обязательно поле "Причина".

При одобрении игры она появляется в публичном каталоге.

При одобрении заявки пользователь получает роль ORGANIZER.

Вкладка "Пользователи" видна только ADMIN.
